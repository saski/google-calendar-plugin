Privacy Policy

Last updated: January 8, 2026

Introduction

Google Calendar Full-Year View ("the Extension") is a Chrome browser extension that enhances Google Calendar by adding a full-year view feature. This privacy policy explains how the Extension handles your data.

Data Collection

The Extension accesses your Google Calendar data to display events in the full-year view. Specifically, the Extension:

  • Reads calendar events from your Google Calendar account
  • Retrieves event details (title, date, time, description) for display purposes
  • Uses Google OAuth 2.0 for secure authentication

The Extension does not collect, store, or transmit any personal information to external servers or third parties.

Data Usage

Calendar data is used solely for display purposes within the Extension. The Extension:

  • Displays your calendar events in the full-year view interface
  • Processes event data locally in your browser
  • Does not modify, delete, or create calendar events
  • Does not share data with any external services

Data Storage

The Extension stores the following data locally in your browser using Chrome's storage API:

  • OAuth Access Token: Stored securely by Chrome to maintain your Google Calendar authentication. This token is required to access your calendar data and is managed by Chrome's identity API.
  • User Preferences: Any extension settings or preferences you configure (if applicable) are stored locally in your browser.

No calendar data is permanently stored. Event data is retrieved from Google Calendar API in real-time and displayed temporarily in the browser interface.

Third-Party Services

The Extension uses the following third-party services:

  • Google Calendar API: The Extension uses Google's Calendar API to read your calendar events. Your use of this Extension is subject to Google's Privacy Policy and Google API Services User Data Policy.

The Extension requests read-only access to your calendar data. You can revoke this access at any time through your Google Account settings.

Permissions

The Extension requires the following permissions:

  • identity: Required for Google OAuth authentication
  • storage: Required to store OAuth tokens and user preferences locally
  • activeTab: Required to interact with Google Calendar pages
  • Host permissions: Access to calendar.google.com and googleapis.com for API calls

Data Security

The Extension implements the following security measures:

  • Uses Google's official OAuth 2.0 authentication flow
  • All API communications use HTTPS encryption
  • OAuth tokens are stored securely by Chrome's identity API
  • No data is transmitted to external servers
  • All processing occurs locally in your browser

Your Rights

You have the following rights regarding your data:

  • Access: You can view all data the Extension accesses through Google Calendar
  • Revoke Access: You can revoke the Extension's access to your calendar at any time through your Google Account settings
  • Uninstall: Uninstalling the Extension removes all locally stored data
  • Delete Data: Clear Chrome's storage to remove Extension data

Children's Privacy

The Extension is not intended for users under the age of 13. We do not knowingly collect personal information from children under 13. If you are a parent or guardian and believe your child has provided us with personal information, please contact us so we can delete such information.

Changes to This Privacy Policy

We may update this privacy policy from time to time. We will notify you of any changes by:

  • Updating the "Last updated" date at the top of this policy
  • Posting a notice in the Extension (for significant changes)

You are advised to review this privacy policy periodically for any changes. Changes to this privacy policy are effective when they are posted on this page.

Open Source

This Extension is open source. The source code is available for review, which allows you to verify how the Extension handles your data. You can review the source code to confirm that:

  • No data is transmitted to external servers
  • All processing occurs locally
  • Only necessary permissions are requested
